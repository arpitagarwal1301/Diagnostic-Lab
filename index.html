  1. INTRODUCTION

1.1 Purpose:-
In our daily life it is a General Practice to go Doctor’s clinic and generally doctor Prescribe some Medical Test. Then we go to a Medical Lab Diagnostic Center.

Medical Lab Diagnostic Center is a place where all kind of medical test conducted. Medical Lab Diagnostic Center always feels requirement of a Software system which help in keeping records and reporting.

Thus Medical Lab Diagnostic Center Management system is very helpful to overcome the general problem.

1.2 Scope:-  Maintaining the masters like test department, category, Group, Test name.  Maintaining the Patient Record.  Maintaining Patient Result.  Maintaining Patient Report.  Maintaining commission of Doctor.  Maintaining report of doctor’s commission.

  1. General Description

2.1 Project perspective The initially this module is proposed to be developed as an independent module. It is proposed to link it with the patient/doctor evaluation module, commission information system, and report generation system in future.

2.2 Product Functions Some of the important functions proposed to be developed are: • Maintaining the masters like test department, category, Group, Test name.

• Maintaining the Patient Record.

• Maintaining Lab Master Data • Maintaining Doctor Master Data Maintaining Test Department Master Data

2.3 User Characteristics The main users of the system are Administrator, Patient, and Doctor. System Administrator: The administrator is responsible for managing database, user management, deletion (which is not allowed to other type of users). An administrator has all rights in every module. Patient: The Faculty can view all the information (Test Details, Test rate, Department details etc). The faculty can also receive the the test report. Doctor:-The doctor send the patient to the diagnostic center and receive the commission from the Lab..

2.4 General constraints • Should Run on the Windows XP/Windows 2000 Platform. • Number of Test should not exceed the capacity of the maximum capacity.. • Test availability.

3.1. The Form which are made in the project are as follows:-

Login form:- This form is used for administrative login. The user enters the username and password, if username and password matched with the database then Administrator Home Page will show.

(2)Administrator Home:- This page is used for nagivation purpose. If we click on any button then particular form will display.

(3)Add Department:- When user enters the Department name and click on ok button then department will be added in the database.

(4) Add Category:- When user selects any department from the list and enter the category then category will be inserted in selected department.

(5) Add Group:- When user selects department and category then enter the group name then group will be added in category.

(5)Add Test:- When user select department, category, test group then enter the test-name, test-rate, test normal value and click on ok button then add will add in database in particular category.

(6)Add Doctor:- This form is used for doctor master entries.User enter doctor name and commission Percentage and click on ok button ,then doctor will save in masters.

(7)Patient Registration:- User fill patient’s personal information, test information and select referred doctor and click on submit button then patient becomes registered.

8) Report by Doctor: This page is used to accept the result of the test conducted on the patient. It is provided by the doctor.

(9)Patient Report:- This page is used to submit the Patient ID and name whose report is to be seen.

This page displays the report of the patient whose ID and name was submitted in the previous page.

(10)Doctor Commission Report:- This page will show the report of doctor’s list with their name, commission per, commission amount.

11) Modify Fields: User can change (edit) the names of existing departments, doctors using this page. It also lets the user to create new account for operating.
